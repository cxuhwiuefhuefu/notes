// 什么是数据库
//     数据的仓库 提供的功能就是对数据的存储/查询/增加/删除/修改 
//     在市场的数据库非常多的 比如oracle(甲骨文) sqlserver(微软) mysql(刚开始是瑞典开发出的一个软件 现在归oracle下面了)
//
// 为什么mysql数据库
//     因为在市场上占有量比较大 占用比较大原因我们简单理解的话 有两个非常非常好的特点 小 一个oracle数据库你去安装的话现在最新版本得达到1G-2G
//         但一个mysql仅仅几十兆 所以说它非常小 第一个它占空间非常小  第二个最大的最大的优点也是使用它的原因所在 免费 
//         让整个软件行业的开发成本降低了 mysql数据库还是个开源的软件 开源的意思就是它的源代码是公开的 允许任何人去进行读取它的源代码
//
// 比如我们现实生活的仓库你去存放物品的时候 这个仓库里边你可以进行乱堆乱放 也可以按照一定的规则进行摆放 如果是一个乱堆乱放的仓库的话
//     你可以想象一下你去里边去找一个东西 非常麻烦  相当于一个垃圾堆一样  但是如果说是一个排放规则的仓库的话 你向里边查找东西是非常方便的
//     你对它进行任何操作都是很方便的 这里边mysql数据同样不允许我们对数据的查找/修改等一系列的操作是麻烦的 当然我们下来用起来越简单越好 
//     这里边mysql数据库对数据的存储也有一定的规划 也就是说有一定的数据结构在这里边放着 那我们现实时候使用office软件的时候 
//     你想象一下当你去做一个数据的时候 你想给别人做数据展现 我们经常采用表格的形式 那同样在mysql数据库中 它进行数据存放也采用的是表格的形式  
//     我们今后再提到mysql数据库里边数据的时候 我们会提到那张那张表 其实是一张又一张的表格在我的数控中存栈 有了表格这东西
//     我就要找到它存在的行和列 其实跟我们现实生活中的表格一样 比如我们拿一个学生的信息进行登记  你也会想象到拿一个学生的信息去登记的时候
//     比如插入一个表格  我们在竖着每一列登记的信息都是一样 第一个位置可以登记学号 第一个位置可以登记姓名 性别/年纪/年龄等等
//     在一行存储的信息就是一个学生的完整信息 那我们在数据库中一般讲到的是字段 记录什么是一个字段??  其实就代表的是我们一个列表示含义 
//     比如说我们的学号字段 我这一列里边某一个取出的记录就是一个学号字段 一行的一个完整信息可以称之为一个记录
//     比如说我一个学生的完整记录它的学号是什么它的姓名是什么它的性别是什么... 那这样的一个表格就是我们的mysql数据库当中存储信息的这么一个格式存在
//
